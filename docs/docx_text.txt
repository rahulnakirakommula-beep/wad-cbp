1.2 Route Guard Behavior (UI Perspective)
Guards are evaluated before any page content renders. During guard evaluation a full-screen loading state is shown 
(see Global Layout Â§2.3). Guards run top-down; the first failing condition wins.
No JWT present
Redirect to /login immediately. No flash of content. No error toast.
JWT present but expired
Redirect to /login. Show toast: 'Your session expired. Please log in again.'
JWT present, email unverified
Redirect to /login. Show toast: 'Please verify your email first.' Offer resend link in toast.
JWT present, onboarding incomplete
Redirect to /onboarding. No toast.
JWT present, user suspended
Redirect to /suspended. Show suspension notice screen.
Correct JWT but wrong role
Redirect to role-appropriate home. Show toast: 'You don't have access to that page.'

2.6 Global Toast System
Toasts are the primary feedback mechanism for async operations. All toasts share consistent behavior.
Toast types

Anatomy

Auto-dismiss
Success: 4000ms. Info: 5000ms. Warning: 6000ms. Error: stays until manually dismissed.
Dismiss on click
Slides out to the right + fades. Duration: 200ms, ease-in.
Stacking
Up to 3 toasts visible. New toasts push others up (200ms reflow animation). Oldest removed first if 4th arrives.
Mobile positioning
Full-width minus 32px horizontal margin. Anchored to bottom of screen, above bottom nav.
Hover pause
Hovering a toast pauses its auto-dismiss countdown. Countdown resumes on mouse-leave.

2.8 Global Error Boundary
If a screen-level JS error occurs, the error boundary catches it and renders a fallback UI â€” a centered card with an 
error illustration, a message ('Something went wrong on this page'), and two actions: 'Reload this page' and 'Go to 
home'. The error is not shown in detail to the user. The navigation remains fully functional.

5.3 Global Animation Rules
NO motion should begin before user intent is clear. Hover animations only trigger after the cursor has rested for 
â‰¥80ms (prevent noise on fast mouse moves).
Skeleton screens appear immediately on load (0ms). Content transitions into skeleton with fade (150ms) only when 
content is ready â€” never before.
Reduce-motion: all animations except loading spinners and skeleton shimmers should be disabled when the 
prefers-reduced-motion media query is active. Critical state changes (errors, successes) should still use non-motion 
feedback (color, icon, text).
Never chain >2 sequential animations visible to the user. Multiple parallel animations on separate elements are 
acceptable.
All lists that reorder should do so with FLIP animation (First, Last, Invert, Play). Elements slide to new positions 
rather than disappearing and reappearing.

Cron runs during admin save. Version is checked before cron writes (cron uses upsert without version check â€” only 
admin saves are version-gated). Admin save may succeed on top of cron's change, or fail on version conflict. If fail:

Very long listing title (>80 chars)
Card: 2-line clamp + ellipsis. Full title in tooltip on hover. Curation panel: full display (no clamp in form input). 
Listing detail: full display.
Very long organisation name (>50 chars)
Card org name: 1-line clamp + ellipsis. Full in tooltip.
Very long description (>5000 words)
Listing detail: full render with no truncation. A 'Jump to top' floating button appears after user scrolls >400px.
Many DomainTags on a card (>3)
First 3 shown, '+N' overflow chip. Clicking '+N' expands to show all inline.
Many linked listings in guide (>5)
Related Opportunities section shows first 5 + 'Show all N' expandable.
Very long note in UserActivity
Notes textarea: scrollable internally. No display length limit. Displayed in activity row as 1-line clamp.
Long org name in source manager table
1-line clamp. Tooltip on hover.
Audit log diff with many fields
Diff cell shows 'N fields changed'. Expanded row shows all.

8.7 Browser / Session Edge Cases
JWT expired mid-session
Next authenticated API call returns 401. Global interceptor catches this. Clears localStorage JWT. Toast: 'Session 
expired. Please log in again.' Navigate to /login after 2s.
Multiple tabs open
If user logs out in one tab, other tabs will fail their next API call with 401 and trigger the expired-session flow.
Browser back after completing onboarding
/onboarding guard: onboardingComplete:true â†’ redirects to /app/feed immediately.
Form submitted with autofill values
Autofill-populated fields fire onChange events correctly. No special handling needed.
Slow typing in search (admin)
Debounce ensures search API call only fires 300ms after last keystroke. Intermediate states show current table rows 
without flickering.
Page refresh on admin curation panel
If form was dirty, data is LOST (no localStorage persistence on admin forms â€” too complex). User sees fresh saved 
state.
Print view
No special print styles in V1. Browser default print behavior.
8.8 Accessibility Edge Cases
Screen reader on listing card
Card container is a single tabbable element with aria-label combining title + org + type + deadline. Save and Ignore 
buttons are separately tabbable with descriptive aria-labels.
Screen reader on status badges
All badges have aria-label with full text. e.g., aria-label='Status: Open'.
Keyboard-only navigation of tag picker
Each chip is tabbable. Space/Enter toggles selection. 'Max reached' message announced to screen reader via aria-live.
High contrast mode
All borders visible. Focus rings maintain â‰¥ 3:1 contrast ratio. No information conveyed by color alone.
Focus after modal close
Focus returns to the element that triggered the modal (stored as a ref before modal opens).
Dynamic content (toasts)
Toast region has aria-live='polite'. Error toasts use aria-live='assertive'. Screen readers announce all toasts.
